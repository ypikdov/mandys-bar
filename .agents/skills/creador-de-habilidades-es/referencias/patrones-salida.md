# Patrones de Salida y Estándares de Calidad

Esta guía asegura que las habilidades produzcan resultados con un formato consistente y alta calidad.

## Plantillas de Formato

Cuando una habilidad deba producir un tipo específico de documento (como un reporte o un README), proporciona una plantilla clara:

```markdown
# [Título del Reporte]

## Resumen Ejecutivo

[Inserta aquí un resumen de 2 párrafos...]

## Hallazgos

- **Hallazgo 1**: Descripción...
- **Hallazgo 2**: Descripción...
```

## Listas de Verificación de Calidad

Instruye a Claude a verificar su propio trabajo antes de entregarlo:

- [ ] ¿El tono es profesional y técnico?
- [ ] ¿Se han seguido todas las reglas de negocio de la sección [X]?
- [ ] ¿Se han incluido todos los campos obligatorios?

## Estilo de Código y Documentación

Especifica estándares para cualquier código generado:

- Usa nombres descriptivos en español para lógica de negocio.
- Sigue los principios S.O.L.I.D.
- Asegúrate de que los comentarios sean útiles y no redundantes.
